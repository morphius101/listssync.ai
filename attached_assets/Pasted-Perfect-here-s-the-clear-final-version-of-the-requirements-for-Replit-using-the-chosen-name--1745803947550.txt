## 🧹 What ClearCheck Will Do (MVP Scope)

**For Property Owners (Admin)**:
- Create checklists easily
- Edit existing tasks (add/delete/require photo)
- Send a link to the cleaner (no login for cleaner needed)
- See real-time progress on tasks
- See uploaded photos attached to tasks
- Download or view full checklist+photos after completion

**For Cleaners (Guest Link View)**:
- Open link from phone (no app download)
- See a checklist of tasks
- Tap to check a task as complete (saved immediately)
- Upload/take photo for a task (if required)
- Add optional remarks at end
- Submit when finished (status changes for owner)

---

## 📱 Mobile-First UX Focus
- Must load in under **1.5 seconds** on 4G.
- Big tap targets (easy checkmark taps and upload buttons).
- Instant visual feedback after checking a task.
- Smooth photo upload experience (either from camera or device gallery).

---

## 🔥 Core Functional Features

| Feature | Details |
|:---|:---|
| Owner Dashboard | Create/Edit Checklists, View Real-time Progress |
| Checklist Management | Add/Edit/Delete tasks, Require/Optional Photo toggle |
| Checklist Link Generation | Create unique links tied to each checklist |
| Cleaner Link View | Mobile checklist view with checkboxes, photo upload, remarks |
| Real-time Updates | Changes show on Owner dashboard within 5–10 seconds |
| Offline Mode (Stretch Goal) | Allow checklist caching locally, auto-sync later if disconnected |
| Report View/Export | After submission, allow viewing or downloading of completed checklist + photos |

## 🛠️ Tech Stack and Structure

**Frontend**:  
- React.js (Vite or Create React App)  
- Tailwind CSS for fast mobile-first styling  
- React Router for handling `/admin` and `/checklist/:id`

**Backend**:
- Firebase Firestore (for real-time checklist data)
- Firebase Storage (for uploaded task photos)

**Hosting**:
- Replit first (easy preview & instant mobile testing)
- Later can migrate to Vercel for custom domains if desired

**Folder Structure:**
```
/src
  /components
    AdminDashboard.jsx
    ChecklistView.jsx
    TaskItem.jsx
    ChecklistEditor.jsx
  /services
    firebase.js
    checklistService.js
App.js
index.js
/public
  index.html
  logo.svg
```

**Firestore DB structure**:
```
/checklists
  /[checklistId]
    ownerId: [owner's id]
    tasks: [
      {
        taskId,
        description,
        completed (bool),
        photoRequired (bool),
        photoUrl (string, optional)
      }
    ]
    remarks: (string)
    createdAt: timestamp
    updatedAt: timestamp
```

**Storage Path**:
```
/photos/[checklistId]/[taskId]/photo.jpg
```

## 🎯 Success Criteria for MVP
- Property owner can create, edit, and send checklists easily.
- Cleaner can open link without downloading anything.
- Real-time (or near-real-time) progress shows on Owner dashboard.
- Photo proof uploads for required tasks.
- MVP ready to demo on Replit without critical bugs.

4. Build **Cleaner Link View** (live checklist progress).
5. Implement **photo uploads** attached to checklist tasks.
6. (Optional) Enable **offline persistence** if simple.
7. Deploy MVP for real-world testing.
